# Content Ownership Matrix

This document defines content ownership and responsibility for different sections of the JASIQ Labs website.

## Overview

The JASIQ Labs website is divided into multiple sections, each managed by different teams. This matrix ensures clear accountability and prevents content conflicts.

## Content Ownership Table

| Section/Page | Owner Team | Content Manager | Update Frequency | Notes |
|-------------|------------|-----------------|------------------|-------|
| **Main Site** | | | | |
| `/` (Home) | Marketing | Marketing Lead | Weekly | Main landing page |
| `/about` | Marketing | Marketing Lead | Monthly | Company information |
| `/contact` | Marketing | Marketing Lead | As needed | Contact information |
| `/products` | Marketing | Marketing Lead | Weekly | Product listings |
| **RealWork Studio** | | | | |
| `/realworkstudio` | Training Team | Training Lead | Weekly | Training division homepage |
| `/realworkstudio/*` | Training Team | Training Lead | Weekly | All training-related pages |
| **TechWorks Studio** | | | | |
| `/techworksstudio` | Tech Team | Tech Lead | Weekly | Technology division homepage |
| `/techworksstudio/*` | Tech Team | Tech Lead | Weekly | All tech-related pages |
| **Shared Resources** | | | | |
| SEO Settings | Marketing | Marketing Lead | Monthly | Global SEO configuration |
| Lead Forms | Marketing | Marketing Lead | As needed | Form configurations |
| Analytics | Marketing | Marketing Lead | Ongoing | Analytics and tracking |

## Content Update Process

1. **Request**: Content owner requests update through designated channel
2. **Review**: Marketing team reviews for brand consistency
3. **Approval**: Content manager approves changes
4. **Implementation**: Development team implements approved changes
5. **Publish**: Content goes live after QA

## Content Guidelines

- All content must align with JASIQ Labs brand guidelines
- SEO best practices must be followed for all public pages
- Content should be reviewed quarterly for accuracy
- Emergency updates can bypass normal process with approval
